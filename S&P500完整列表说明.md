## ⚠️ 关于完整500个股票

### 为什么不直接使用全部500个？

1. **API限制**：
   - 免费API：每天500次请求
   - 500个股票 = 500次请求 = **一次就用光配额**
   - 刷新一次页面就会超过限制，之后只能使用模拟数据

2. **性能问题**：
   - 加载500个股票需要很长时间
   - 浏览器渲染500行数据会很慢
   - 用户体验差

3. **实用性**：
   - 对于编码挑战，50-100个股票已经足够
   - 展示功能比股票数量更重要

### 如果你确实需要更多股票

**方案A：使用100-150个股票（平衡方案）**
```typescript
const DEMO_STOCKS = getSP500Stocks(100); // 或 getSP500Stocks(150)
```
- 涵盖更多公司
- 仍可刷新几次
- 性能可接受

**方案B：实现分页功能（高级）**
- 每次加载50个股票
- 用户可以翻页查看
- 需要额外开发

**方案C：使用模拟数据模式**
- 如果API配额用完，自动使用模拟数据
- 可以显示任意数量的股票
- 但数据不是真实的
