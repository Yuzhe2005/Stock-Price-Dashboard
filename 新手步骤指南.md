## 第一步：检查是否安装了 Node.js

### 为什么要做这一步？
Node.js 是一个运行 JavaScript 的工具，我们的项目需要它来安装依赖和运行代码。

### 怎么做？

1. **打开命令行（终端）**
   - 按 `Windows键 + R`
   - 输入 `cmd` 或 `powershell`
   - 按回车

2. **检查是否已安装 Node.js**
   在命令行中输入：
   ```bash
   node --version
   ```
   
   **如果显示版本号**（例如 `v18.17.0`）：
   - ✅ 已安装！跳到第二步
   
   **如果显示 "不是内部或外部命令"**：
   - ❌ 需要安装 Node.js

3. **如果没有安装，下载安装 Node.js**
   - 访问：https://nodejs.org/
   - 下载 "LTS" 版本（长期支持版本，更稳定）
   - 双击下载的文件，一路点"下一步"安装
   - 安装完成后，**重启命令行**，再次运行 `node --version` 确认

## 第三步：安装项目依赖

### 为什么要做这一步？
项目依赖就像"零件清单"，我们的项目需要很多"零件"（代码库）才能运行：
- React（用来构建网页界面）
- TypeScript（让代码更安全）
- Tailwind CSS（用来美化界面）
- Vite（用来快速开发和构建）

`npm install` 命令会从网上下载并安装所有这些"零件"。

### 怎么做？

1. **在项目文件夹中运行安装命令**
   ```bash
   npm install
   ```
   
   **这个过程需要几分钟**，你会看到很多信息在滚动，这是正常的！

2. **等待完成**
   - 当看到类似 `added 500 packages` 的信息时，表示安装成功
   - 如果看到错误信息，告诉我，我会帮你解决

3. **检查是否成功**
   安装完成后，文件夹里会出现一个 `node_modules` 文件夹
   - 这个文件夹包含了所有安装的"零件"
   - **注意**：这个文件夹很大，不需要手动打开

## 第四步：启动开发服务器

开发服务器就像一个"测试环境"，可以让你在浏览器中看到你的项目运行效果。每次修改代码，浏览器会自动刷新显示最新效果。

## 第六步：构建生产版本

### 为什么要做这一步？
开发服务器只是用来测试的，不能给别人访问。我们需要"打包"代码，创建一个可以在任何地方运行的版本。这个过程会：
- 优化代码（让文件更小、运行更快）
- 检查错误
- 生成可以部署的文件

### 怎么做？

1. **停止开发服务器**（如果还在运行）
   - 按 `Ctrl + C`

2. **运行构建命令**
   ```bash
   npm run build
   ```

3. **等待完成**
   当看到 `✓ built in 5.23s` 这样的信息时，表示成功

4. **检查构建结果**
   项目文件夹里会出现一个新文件夹 `dist`
   - 这个文件夹包含了可以部署的所有文件
   - **不要手动修改这个文件夹**

## 第七步：准备部署到 GitHub

### 为什么要做这一步？
GitHub 是一个代码托管平台，我们需要把代码上传到 GitHub，这样才能部署到 Vercel 或 Netlify。

### 怎么做？

#### 7.1 创建 GitHub 账号（如果还没有）

1. 访问：https://github.com
2. 点击 "Sign up" 注册账号
3. 验证邮箱

#### 7.2 创建新的 GitHub 仓库

1. 登录 GitHub 后，点击右上角的 "+" 号
2. 选择 "New repository"
3. 填写信息：
   - **Repository name**: `stock-price-dashboard`（或你喜欢的名字）
   - **Description**: `股票价格仪表板 - React + TypeScript + Tailwind CSS`
   - **选择 Public**（公开，这样部署更方便）
   - **不要**勾选 "Initialize this repository with a README"（我们已经有文件了）
4. 点击 "Create repository"

#### 7.3 初始化 Git（如果还没初始化）

在项目文件夹中运行：

```bash
git init
```

#### 7.4 添加所有文件到 Git

```bash
git add .
```

#### 7.5 提交代码

```bash
git commit -m "Initial commit: Stock Price Dashboard"
```

#### 7.6 连接到 GitHub 仓库

**重要**：把下面的 `YOUR_USERNAME` 替换成你的 GitHub 用户名

```bash
git remote add origin https://github.com/YOUR_USERNAME/stock-price-dashboard.git
```

例如，如果你的用户名是 `zhangsan`，命令应该是：
```bash
git remote add origin https://github.com/zhangsan/stock-price-dashboard.git
```

#### 7.7 推送代码到 GitHub

```bash
git branch -M main
git push -u origin main
```

**注意**：第一次推送需要登录 GitHub，可能会弹出登录窗口。

## 第八步：部署到 Vercel（推荐）

### 为什么要做这一步？
Vercel 是一个免费的网站托管平台，可以让全世界的用户访问你的网站。它操作简单，而且对 React 项目支持很好。

### 怎么做？

1. **访问 Vercel**
   - 打开：https://vercel.com

2. **登录**
   - 点击 "Sign Up"
   - 选择 "Continue with GitHub"（使用 GitHub 账号登录）

3. **创建新项目**
   - 登录后，点击 "Add New..." -> "Project"
   - 在 "Import Git Repository" 中，找到你的 `stock-price-dashboard` 仓库
   - 点击 "Import"

4. **配置项目**
   - **Framework Preset**: Vite（应该自动检测到）
   - **Root Directory**: `./`（默认）
   - **Build Command**: `npm run build`（应该已经自动填好）
   - **Output Directory**: `dist`（应该已经自动填好）
   - 其他保持默认

5. **部署**
   - 点击 "Deploy" 按钮
   - 等待 1-2 分钟，部署会自动完成

6. **获取你的网站地址**
   - 部署完成后，你会看到一个绿色的 "Visit" 按钮
   - 点击后可以访问你的网站
   - 你的网站地址类似：`https://stock-price-dashboard-xxx.vercel.app`

**🎉 恭喜！你的网站已经上线了！**

## ❓ 常见问题解答

### Q1: 运行 `npm install` 时很慢怎么办？

**A**: 这是正常的，因为需要下载很多文件。可以：
- 等待完成（通常 3-5 分钟）
- 如果卡住，按 `Ctrl + C` 停止，然后重新运行

### Q2: 运行 `npm run dev` 后浏览器显示错误？

**A**: 可能的原因：
1. 端口 5173 被占用：关闭其他可能使用这个端口的程序
2. 代码有错误：告诉我具体的错误信息

### Q3: Git 命令不识别怎么办？

**A**: 需要安装 Git：
1. 访问：https://git-scm.com/download/win
2. 下载并安装
3. 重启命令行

### Q4: Vercel 部署失败怎么办？

**A**: 检查以下几点：
1. GitHub 仓库是否成功推送代码
2. `package.json` 中是否有 `build` 脚本
3. 查看 Vercel 的错误日志，告诉我具体错误信息

### Q5: 网站可以访问，但数据不显示？

**A**: 这是正常的！当前使用的是模拟数据。如果要使用真实数据：
- 需要注册股票 API（Finnhub 或 Alpha Vantage）
- 获取 API Key
- 修改代码中的 API 配置
